# EPC Workbench 확장 로드맵 (Beginner → Master) v0.1

작성일: 2026-03-07  
목표: 현재 저장소의 강점을 유지하면서, **초심자 친화성 + 시각화 강화 + 고급 사용자 생산성**을 동시에 올리는 실행안 제시

## 1) 현재 상태 요약 (폴더 리뷰 기반)

### 강점
- 7개 공종 워크벤치, 4계층 검증, 기준/법령 추적 구조가 이미 구현됨.
- 프론트엔드에 시각화 컴포넌트가 공종별로 분리되어 확장성이 좋음.
- `calculation-guide`, `glossary` 페이지가 있어 교육형 UX의 기반이 있음.
- 백엔드/프론트/테스트/품질게이트 스크립트가 갖춰져 운영 가능한 형태.

### 개선 여지
- 저장소 용량이 매우 큼(`frontend`, `datasets`, `docs` 중심) → 기여/온보딩 비용 상승.
- 초심자가 “어디서 시작하고 무엇을 봐야 하는지”에 대한 단계형 학습 플로우는 부족.
- 고급 사용자를 위한 비교/시나리오 분석/배치 실행 UI는 제한적.
- 시각화는 존재하지만, “의사결정 중심(왜 위험한지/어떻게 바꿔야 하는지)” 스토리텔링 강화 필요.

---

## 2) 제품 확장 방향 (핵심 3축)

## A. Beginner 모드 (입문자도 바로 이해)
1. **Guided Run(단계형 실행)**
   - 입력 → 계산 → 검증 → 조치결정까지 Wizard 방식 제공
   - 각 단계에 “왜 필요한 입력인지” 한 줄 설명/예시 자동 노출
2. **학습 오버레이**
   - 결과 카드에 “한글 쉬운 설명 / 엔지니어 설명” 토글
   - 위험도 신호등의 의미와 다음 액션 템플릿 제공
3. **10분 온보딩 시나리오**
   - `/calculation-guide`에 공종별 샘플 1개씩 “클릭 실행형 튜토리얼” 추가

## B. Visualization 강화 (결정이 빨라지는 화면)
1. **원인-결과 체인 차트**
   - 입력 변수 변화가 위험도/수명/검사주기에 미치는 영향 화살표 시각화
2. **Cross-Discipline Impact Map**
   - 공종 간 영향(예: 배관↔정기기)을 네트워크 그래프로 표시
3. **Time-Series + Threshold Overlay**
   - 과거 이력, 현재값, 기준치, 예측치(다음 점검 시점) 동시 표시

## C. Master 모드 (숙련자 최고의 도구)
1. **Scenario Lab**
   - 변수 2~5개를 동시에 바꿔 여러 시나리오를 한 번에 비교
2. **Batch Screening**
   - 자산 CSV 업로드로 다건 일괄 계산 + 위험 우선순위 자동 정렬
3. **Evidence Pack Export**
   - 계산 근거, 검증 로그, 관련 표준/법령 링크를 묶어 감사용 패키지 생성

---

## 3) 실행 우선순위 (90일)

### Phase 1 (0~30일) — 초심자 진입장벽 제거
- Guided Run 1차(배관/정기기 우선)
- 결과 카드 쉬운 설명 토글
- 시작 페이지에 “역할별 시작 버튼(입문/실무/검증)”

### Phase 2 (31~60일) — 시각화 중심 의사결정 UX
- Time-Series + Threshold Overlay 공통 컴포넌트화
- Cross-Discipline Impact Map MVP
- 위험 원인 상위 3개 자동 하이라이트

### Phase 3 (61~90일) — Master 생산성 모드
- Scenario Lab
- Batch Screening
- Evidence Pack Export v1 (Markdown + JSON + 링크 인덱스)

---

## 4) 코드/구조 관점 권장 작업

1. **Repository hygiene**
   - 대용량 산출물(`logs`, 빌드 산출물, 중복 데이터)의 아카이브/정리 정책 수립
   - `.gitignore` 강화(캐시/빌드/로그/임시 파일)
2. **Frontend 정보 구조 정리**
   - Beginner/Standard/Master UI 모드를 라우트 또는 상단 토글로 명확히 분리
3. **공통 시각화 컴포넌트 추상화**
   - 공종별 차트 중복 로직을 공통 렌더러로 통합
4. **성과 측정 지표 도입**
   - 첫 실행 완료율, 평균 판단 시간, 경고 처리 리드타임, 재실행률

---

## 5) 성공 기준 (측정 가능한 KPI)
- 초심자 첫 실행 완료율: +30%
- 실행 후 “조치 결정까지 시간”: -25%
- 다건 분석(10건 이상) 처리시간: -40%
- 감사/보고서 작성 시간: -35%

3. `Scenario Lab` 데이터 계약(입력/출력 스키마) 먼저 정의
